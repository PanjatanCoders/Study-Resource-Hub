⚡ Essential DAX Functions (The Holy Grail)

Text Functions

TEXT
CONCATENATE()
CONCATENATE(text1, text2)
Joins two text strings into one. Period.
TEXT
LEFT() / RIGHT() / MID()
LEFT(text, num_chars)
RIGHT(text, num_chars)
MID(text, start, num_chars)
Extract parts of text strings like a boss.
TEXT
SUBSTITUTE()
SUBSTITUTE(text, old_text, new_text)
Replace text within a string - find and replace on steroids.

Date & Time Functions

DATE
TODAY() / NOW()
TODAY()
NOW()
Get current date or current date/time. Real-time vibes!
DATE
DATEDIFF()
DATEDIFF(start_date, end_date, interval)
Calculate the difference between two dates. Interval can be DAY, MONTH, YEAR, etc.
DATE
DATEADD()
DATEADD(dates, number_of_intervals, interval)
Shift dates by a specified time period. Time travel in DAX!
Use CALENDAR() or CALENDARAUTO() to create date tables. Your future self will thank you! 📅

🧮 Aggregation Functions (Math is Life)

MATH
SUM() / AVERAGE()
SUM(column)
AVERAGE(column)
Basic math functions. The bread and butter of data analysis.
MATH
SUMX() / AVERAGEX()
SUMX(table, expression)
AVERAGEX(table, expression)
Iterator functions that apply expressions row by row. More flexible than basic SUM/AVERAGE.
MATH
COUNT() / COUNTA() / COUNTX()
COUNT(column)
COUNTA(column)
COUNTX(table, expression)
COUNT numbers, COUNTA non-blanks, COUNTX with expressions.
Remember: SUM() works on columns, SUMX() works on tables with expressions. Don't mix them up!

🎯 Filter Functions (The Game Changers)

FILTER
CALCULATE()
CALCULATE(expression, filter1, filter2, ...)
The GOAT of DAX. Modifies filter context. Use it everywhere!
FILTER
FILTER()
FILTER(table, condition)
Returns a table with rows that meet the condition.
FILTER
ALL() / ALLEXCEPT()
ALL(table_or_column)
ALLEXCEPT(table, column1, column2, ...)
Remove filters from columns or tables. ALL removes everything, ALLEXCEPT keeps specified columns.
CALCULATE() Examples:
CALCULATE(SUM(Sales), Region = "North")
CALCULATE([Total Sales], YEAR(Date) = 2023)
ALL() vs ALLEXCEPT():
ALL(Sales) -- removes all filters
ALLEXCEPT(Sales, Region) -- keeps Region filter

🤖 Logical Functions (Decision Making 101)

LOGIC
IF() / SWITCH()
IF(condition, true_value, false_value)
SWITCH(expression, value1, result1, value2, result2, ...)
IF for simple conditions, SWITCH for multiple conditions. Choose your fighter!
LOGIC
AND() / OR() / NOT()
AND(condition1, condition2)
OR(condition1, condition2)
NOT(condition)
Combine logical conditions. Boolean logic for the win!
LOGIC
ISBLANK() / ISEMPTY()
ISBLANK(value)
ISEMPTY(table)
Check for blank values or empty tables. Handle your nulls like a pro!
Use SWITCH() instead of nested IF() statements for cleaner code. Your colleagues will respect you more! ✊

⏰ Time Intelligence (Time Travel Magic)

TIME
SAMEPERIODLASTYEAR()
SAMEPERIODLASTYEAR(dates)
Compare current period with same period last year. Year-over-year comparisons made easy!
TIME
TOTALYTD() / TOTALQTD() / TOTALMTD()
TOTALYTD(expression, dates)
TOTALQTD(expression, dates)
TOTALMTD(expression, dates)
Running totals for year, quarter, or month to date.
TIME
DATESBETWEEN()
DATESBETWEEN(dates, start_date, end_date)
Get dates within a specific range. Perfect for custom time periods!
Time Intelligence Requirement: You MUST have a proper date table with continuous dates! No shortcuts! 📊

🔗 Relationship Functions (Connect the Dots)

RELATIONSHIP
RELATED() / RELATEDTABLE()
RELATED(column)
RELATEDTABLE(table)
RELATED gets values from the "one" side, RELATEDTABLE from the "many" side.
RELATIONSHIP
USERELATIONSHIP()
USERELATIONSHIP(column1, column2)
Activate inactive relationships. For when you have multiple date relationships!
RELATIONSHIP
LOOKUPVALUE()
LOOKUPVALUE(return_column, search_column, search_value)
Like VLOOKUP but better. Find values across tables without relationships.
RELATED() and RELATEDTABLE() only work when there's an active relationship. Check your model first!

💡 Common DAX Patterns (The Greatest Hits)

Running Totals

Running Total = CALCULATE( SUM(Sales[Amount]), FILTER( ALL(Sales[Date]), Sales[Date] <= MAX(Sales[Date]) ) )

Ranking

Product Rank = RANKX( ALL(Products[Product]), CALCULATE(SUM(Sales[Amount])), , DESC )

Previous Period Comparison

Previous Month Sales = CALCULATE( SUM(Sales[Amount]), DATEADD(Calendar[Date], -1, MONTH) )

Percentage of Total

% of Total = DIVIDE( SUM(Sales[Amount]), CALCULATE( SUM(Sales[Amount]), ALL(Products[Category]) ) )
Always use DIVIDE() instead of "/" to handle division by zero. It returns BLANK instead of error!

🚀 Pro Tips (Level Up Your Game)

Performance Tips

  • Use variables (VAR) to store repeated calculations
  • SUMX > SUM when you need row-by-row calculations
  • Avoid using FILTER() in CALCULATE() when possible
  • Use KEEPFILTERS() to add filters instead of overriding

Best Practices

  • Name your measures clearly (no cryptic abbreviations!)
  • Use proper formatting in your DAX code
  • Comment your complex formulas
  • Test edge cases (nulls, zeros, empty tables)

Variable Usage (VAR)

Sales Growth = VAR CurrentSales = SUM(Sales[Amount]) VAR PreviousYearSales = CALCULATE( SUM(Sales[Amount]), SAMEPERIODLASTYEAR(Calendar[Date]) ) RETURN DIVIDE(CurrentSales - PreviousYearSales, PreviousYearSales)
Variables make your code more readable and can improve performance by avoiding repeated calculations!

🐛 Common Issues (Debug Like a Pro)

Circular Dependencies: When measures reference each other. Break the cycle!
Filter Context Issues: Understanding row context vs filter context is crucial!
BLANK vs 0: They're different! Use ISBLANK() to check, use IF() to handle.

Quick Fixes

  • Getting wrong totals? Check if you need SUMX instead of SUM
  • Time intelligence not working? Verify your date table is properly marked
  • RELATED() returning blank? Check your relationships and cardinality
  • Performance issues? Look for unnecessary FILTER() functions
